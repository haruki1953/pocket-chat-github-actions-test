Pinia 持久化与通过 Store 版本号避免类型变更导致的旧数据运行时错误

## 背景问题

在 Vue3 + Pinia 中使用持久化（persist）时，数据会存储在 `localStorage` 。  
如果前端更新了 store 的结构（字段名、类型、默认值），但浏览器里还保留着旧版本的数据，就可能出现：
- 类型不符，导致运行时错误
- 旧字段残留，污染新逻辑
- 调试困难，编译期类型检查无法覆盖运行时数据

## 解决方案：版本号拼接 Store id

通过在 `defineStore` 的 id 中拼接一个 **版本号常量**，可以让 Pinia 在持久化时认为这是一个全新的 store，从而避免加载旧数据。

## 示例代码
```ts
/**
 * 当前 Store 的版本号。
 *
 * 当 store 的结构（字段、类型）发生不兼容更新时，
 * 修改此版本号即可强制生成一个新的持久化存储空间。
 * 避免旧数据与新类型不符导致运行时错误。
 */
const STORE_VERSION = 'v1'

export const useUploadImageStore = defineStore(
  `pocket-together-upload-image-${STORE_VERSION}`,
  () => {
    // ...
  },
  {
    persist: true,
  },
)
```

## 特性
- **强制清理旧数据**：每次修改版本号都会让旧数据失效，避免类型不符。
- **简单直接**：无需复杂迁移逻辑，只需 bump 一下版本号。
- **适合临时性数据**：浏览器端的持久化数据通常不重要，可以安全丢弃。
- **可控升级**：开发者决定何时丢弃旧数据，避免运行时错误。
