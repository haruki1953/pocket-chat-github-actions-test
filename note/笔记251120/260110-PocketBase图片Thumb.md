# 📘 PocketBase Thumb 格式笔记（简明机制版）

PocketBase 的 thumb 参数本质上是一套“服务器端图片缩放/裁剪指令”。  
理解它的关键是：**WxH 系列是 cover，WxHf 是 contain**。

---

## 🎯 1. `WxH` —— cover（从中心裁剪）

示例：`100x300`

- 先把图片按比例放大到**至少覆盖** 100×300 的区域
- 再从**中心**裁剪出 100×300
- **不是按像素直接裁中心**，而是典型的 CSS `object-fit: cover` 行为

适合：头像、封面、固定比例展示位

---

## 🎯 2. `WxHt` / `WxHb` —— cover + 不同的纵向对齐

示例：`100x300t`（top）、`100x300b`（bottom）

它们和 `WxH` 的机制完全一样，都是 **cover**，区别只有：

- **横向永远居中**
- **纵向对齐方式不同**
    - `t` → 从顶部裁
    - `b` → 从底部裁
    - 无后缀 → 从中间裁

适合：

- `t`：人像（保留头部）
- `b`：食物/物品（主体在底部）

---

## 🎯 3. `WxHf` —— contain（完整显示，不裁剪）

示例：`100x300f`

- 将图片缩放到**完全放进** 100×300 的框
- 保持比例
- **不裁剪**
- 可能出现留白（letterbox）

这才是我以前误以为的 “fit inside” 行为。

适合：文档、二维码、截图、不能裁剪的内容

---

## 🎯 4. `0xH` / `Wx0` —— 单边缩放，保持比例

示例：`0x300`、`100x0`

- `0xH`：高度固定，宽度按比例
- `Wx0`：宽度固定，高度按比例
- 不裁剪

适合：聊天图片预览、只关心某一边尺寸的场景

---

# 🧠 总结一句话

- **WxH / WxHt / WxHb = cover（横向居中，纵向不同对齐）**
- **WxHf = contain（完整显示，不裁剪）**
- **0xH / Wx0 = 单边缩放**

这就是 PocketBase thumb 的核心机制。
